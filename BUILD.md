# Building the JavaScript Library

The JavaScript library is located in `resources/js/remarkd/` and is bundled into a single file `resources/js/remarkd.js` for easier distribution.

## Building

To build the bundled JavaScript file:

```bash
npm install
npm run build
```

This will:
1. Install esbuild as a dev dependency
2. Bundle all ES modules from `resources/js/remarkd/` into `resources/js/remarkd.js`
3. Output a global IIFE format file that creates `window.Remarkd` for use in browsers

## Usage

After building, the library is available globally as `window.Remarkd`. You can use it in inline scripts without any imports:

```html
<script src="/resources/js/remarkd.js"></script>
<script>
  // Parse remarkd text
  const html = await Remarkd.parse('= My Title\n\nSome content');
  document.getElementById('output').innerHTML = html;
  
  // Or use the classes directly
  const remarkd = new Remarkd.Remarkd();
  const parser = new Remarkd.Parser(lines, remarkd);
  const doc = await parser.parse();
  const html2 = doc.render();
</script>
```

Available exports:
- `Remarkd.parse(text)` - Quick parse function
- `Remarkd.Remarkd` - Main Remarkd class
- `Remarkd.Parser` - Parser class
- `Remarkd.Document` - Document class
- `Remarkd.Section` - Section class
- `Remarkd.RemarkdContext` - Context class
- `Remarkd.BlockEngine` - Block engine
- `Remarkd.RuleEngine` - Rule engine
- `Remarkd.ObjectEngine` - Object engine
- `Remarkd.Attributes` - Attributes class
- `Remarkd.DocumentData` - DocumentData class
- Plus all blocks, rules, and objects

## Development

The source files are in `resources/js/remarkd/` organized as:
- Core classes: `Remarkd.js`, `Parser.js`, `Document.js`, etc.
- Blocks: `blocks/` directory
- Rules: `rules/` directory  
- Objects: `objects/` directory

Make changes to the source files, then run `npm run build` to regenerate the bundle.

## Notes

- The bundled file is excluded from git (see `.gitignore`)
- The bundle uses ES module format for modern browser compatibility
- Source maps are not generated by default (can be enabled in `build.js`)
